システム概要

Netdoa NX（WebシステムのPOSデータ管理ページ）から全店舗の売上データを毎日自動でダウンロードし、データベースに蓄積するシステムです。蓄積されたデータはダッシュボードで分析できます。

取得対象

全22店舗（店舗コード 1〜22、および100）の商品売上実績 日別データ

自動実行スケジュール

タスクスケジューラに2つのタスクが登録されており、それぞれ自動で実行されます。どちらのタスクも「未取得の日付だけ」を処理するため、重複実行しても問題ありません。

タスク名 実行タイミング 役割 状態
よしや_POS日次処理 毎日 07:00 通常の定期実行。PCが7時に起動していれば自動実行。 有効
yoshiya_POS_logon ログイン時
（毎回）		 7時以降にPCを起動した場合や、数日間電源OFFだった場合にまとめて補完実行。 有効
数日間PCをオフにしていても大丈夫 — 次回PC起動・ログイン時に、不足分をすべて自動で取り込みます。

自動実行の確認方法

スタートメニューで「タスク スケジューラ」を開き、タスク一覧から上記2つのタスクを確認できます。「前回の実行時刻」や「前回の実行結果」が確認できます。

手動実行の方法

自動実行されなかった場合や、今すぐデータを取得したい場合は手動で実行します。

手動実行手順
  1. PowerShell を開く
    スタートメニューで「PowerShell」を検索して起動（管理者不要）。
  2. 作業フォルダーに移動
    cd "C:\one_tenri\OneDrive\HTY\vscode\売上管理\pos_system"
  3. スクリプトを実行
    python run_daily.py
    未取得の日付を自動検出して順番に処理します。すでに最新であれば「未取得の日付なし」と表示されます。
1日分のデータ処理には約2〜5分かかります。複数日分まとめて処理する場合はその分時間がかかります。処理中はPowerShellを閉じないでください。

特定日付だけを取得したい場合

例：2026年6月25日のCSVのみダウンロード
python download.py 20260625

特定のCSVをデータベースに入れ直したい場合

例：20260625.csv を再取込
python import_db.py downloads\20260625.csv

データ・ログ確認

データベースの最新日付を確認する

PowerShellで以下を実行すると、DBに入っている最新日付と直近のデータ件数を確認できます。

python -c " import sqlite3 con = sqlite3.connect('pos_data.db') print('最新日付:', con.execute('SELECT MAX(pos_date) FROM sales').fetchone()[0]) print('総件数:', con.execute('SELECT COUNT(*) FROM sales').fetchone()[0]) con.close() "

ダウンロード済みCSVを確認する

Get-ChildItem downloads\*.csv | Sort-Object Name | Select-Object Name, Length, LastWriteTime

実行ログを確認する

ログファイルは logs\ フォルダーに月別で保存されています。

ファイル名内容
daily_YYYYMM.logrun_daily.py の実行ログ（何日分処理したか、件数など）
download_YYYYMM.logdownload.py のログ（Netdoa NX への接続・ダウンロード状況）
import_YYYYMM.logimport_db.py のログ（DB取込件数・スキップ件数）
最新ログを開く
Get-ChildItem logs\daily_*.log | Sort-Object LastWriteTime -Descending | Select-Object -First 1 | Get-Content -Tail 30

ダッシュボード起動

蓄積されたデータをブラウザで分析するダッシュボードを起動できます。

起動手順
  1. PowerShell を開いて作業フォルダーに移動
    cd "C:\one_tenri\OneDrive\HTY\vscode\売上管理\pos_system"
  2. ダッシュボードを起動
    streamlit run dashboard.py
  3. ブラウザで確認
    自動的にブラウザが開きます。開かない場合は http://localhost:8501 にアクセスしてください。
  4. 終了するとき
    PowerShellで Ctrl + C を押してください。
ダッシュボードを起動したままPCを終了してもデータには影響ありません。ただし次回起動時は再度 streamlit run dashboard.py を実行してください。

トラブル対処

症状 原因 対処
要確認
自動実行されていない（ログに何も残っていない）		 PC が7時頃に電源OFF だった、またはログインしていなかった PowerShellで python run_daily.py を手動実行する。未取得分をまとめて取得します。
エラー
「CSVダウンロード失敗」とログに表示される		 Netdoa NX への接続失敗、またはCSV生成タイムアウト（25分超） インターネット接続を確認してから再実行。繰り返す場合はNetdoa NXに直接ログインして状態を確認してください。
エラー
「ログイン失敗」とログに表示される		 Netdoa NXのパスワードが変わった config.json の password を新しいパスワードに更新してください。
要確認
DBの最新日付が数日前のまま		 自動実行が何日か動いていない python run_daily.py を実行。複数日分をまとめて処理します（1日あたり約3〜5分）。
エラー
Python が見つからないというエラー		 Python がインストールされていない、またはパスが通っていない Python 3.12 を公式サイトからインストールし直してください。「Add to PATH」にチェックを入れてインストール。
要確認
ダッシュボードが起動しない		 streamlit がインストールされていない PowerShellで pip install streamlit を実行してください。
困ったときは — ログファイル（logs\daily_YYYYMM.log）の末尾を確認すると、エラーの詳細が書かれています。

ファイル構成

すべてのファイルは以下のフォルダーにあります：

C:\one_tenri\OneDrive\HTY\vscode\売上管理\pos_system\
pos_system/ ├── run_daily.py 　タスクスケジューラから呼ばれるメインスクリプト ├── download.py 　Netdoa NXからCSVをダウンロード ├── import_db.py 　CSVをデータベースに取り込む ├── dashboard.py 　分析ダッシュボード（Streamlit） ├── config.json 　認証情報・設定（バックアップ注意） ├── pos_data.db 　売上データベース（SQLite） ├── downloads/ 　ダウンロードしたCSVファイル │ └── YYYYMMDD.csv 　　日付ごとのCSV ├── logs/ 　実行ログ │ └── daily_YYYYMM.log　　月別ログファイル └── requirements.txt 　必要なPythonパッケージ一覧
config.json と pos_data.db は自動バックアップされません。 定期的に別ドライブやUSBにコピーしておくことをおすすめします。特に config.json にはNetdoa NXのログイン情報が入っています。
