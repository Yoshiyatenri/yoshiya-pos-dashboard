# メール処理（キーワード検知ツール） 使い方マニュアル

指定したメールアドレスに届くメールから、重要なキーワード（「至急」「締め切り」など）を自動で検知し、
AIが緊急度を判定して、重要なメールが来たことを通知するツールです。
このマニュアルの通りに操作すれば使えるようになっています。

---

## 1. このツールが行うこと

1. 指定したメールアドレス（社内メール）に届くメールを自動で確認する
2. 件名に登録しておいたキーワード（「至急」「締め切り」など）が含まれているかチェックする（本文はチェック対象外です）
3. キーワードに該当したメールについて、Claude AIに「緊急度」と「返信が必要か」を判定させる
4. 該当メールが1件でもあれば、それらを**まとめた通知メール1通**があなたのメールアドレスに届く
5. 該当メールが0件の場合は、何も送られません

**やること（あなたの作業）は「`config.json`で設定」と「実行予約（タスクスケジューラ）」だけです。**

---

## 2. 初回だけ行う準備

### 準備1: `config.json`を設定する

このツールを動作させるために必要な設定ファイルを作成・編集します。

`メール処理/config.json` を開いてください。以下の内容を参考に、括弧内の`ここに...`の部分を実際の値に書き換えて保存してください。

```json
{
  "imap_host": "okashinodepart.net",
  "imap_port": 143,
  "use_ssl": false,
  "user": "tenri@okashinodepart.net",
  "password": "ここにメールパスワードを入力してください",
  "keywords": ["オンライン集計", "締め切り", "至急"],
  "anthropic_api_key": "ここにClaude APIキーを入力してください",
  "smtp_host": "okashinodepart.net",
  "smtp_port": 587,
  "notify_to": "ここに通知を受け取りたいメールアドレスを入力してください"
}
```

各項目の説明:

- `imap_host`, `imap_port`, `use_ssl`: メール受信サーバーの接続設定（既にメールクライアントで設定済みなら、そこから確認してください）
- `user`: メールアドレス
- `password`: メールパスワード
- `keywords`: チェック対象のキーワード。複数指定できます（カンマ `,` で区切ってください）
- `anthropic_api_key`: Claude APIの認証キー。取得方法が分からない場合は次の作業のときに聞いてください
- `smtp_host`, `smtp_port`: 通知メール送信サーバーの設定
- `notify_to`: 検知結果の通知メールを受け取りたいメールアドレス

> ⚠️ **注意**: `config.json` にはパスワードやAPIキーが入るため、絶対に人に見せたりGitHub等にアップロードしたりしないでください（このツールでは自動的にGit管理から除外される設定になっているので、通常の操作をしている限り心配いりません）。

### 準備2: 必要な部品（パッケージ）をインストールする

VS Codeのターミナルを開いて、以下を実行してください（1回だけでOKです）。

```powershell
pip install -r メール処理/requirements.txt
```

「Successfully installed ...」のような表示が出れば完了です。

### 準備3: タスクスケジューラに登録する（自動実行の場合）

毎日決まった時間に自動実行させたい場合は、Windows のタスクスケジューラに登録します。

`メール処理/setup_scheduler.bat` をダブルクリックして実行すると、自動的に登録されます。
登録の詳細は `setup_scheduler.bat` の中身を確認するか、次の作業のときに聞いてください。

## 3. 結果の見方

### 通知メール

キーワードに該当したメールがあった場合、チェック実行のたびに**まとめて1通**、
`config.json`の`notify_to`に設定したアドレスへ通知メールが届きます。該当メールが
1件もない場合は、メールは送信されません。

メール本文には、該当したメールごとに以下の内容が記載されます。

- 差出人
- 件名
- ヒットキーワード
- 緊急度（AIが判定: 高・中・低）
- 返信要否（AIが判定: 要・不要）
- 理由（AIが判定した根拠）

### `mail_check.log`（実行ログ）

接続エラーや、通知メール送信に失敗した場合の詳細（該当メールの件名・緊急度等）が
記録されます。普段は見なくて大丈夫です。

## 5. よくある質問・トラブル対応

### Q. 実行すると英語で「FileNotFoundError: ... config.json」と出る

`メール処理` フォルダの中にいない状態でツールを実行しています。ターミナルで `cd メール処理` を実行してから、もう一度 `python mail_check.py` を試してください。

### Q. 実行すると英語で「json.decoder.JSONDecodeError」と出る

`config.json` を手で編集した際に、カンマの付け忘れ・付けすぎなどでファイルが壊れています。設定値を追加するときは、以下のように**項目と項目の間にカンマ `,` を入れる**のを忘れないでください（一番最後の項目の後ろにはカンマを付けません）。

```json
{
  "key1": "値1",
  "key2": "値2"
}
```

### Q. 通知メールの「緊急度」「返信要否」が「AI判定失敗」になっている

`config.json` の `anthropic_api_key` が未設定（プレースホルダーのまま）か、間違っている可能性があります。準備1の手順でキーを設定し直してください。この場合でもキーワード検知自体は行われ、通知メールは届きます（AI判定の欄だけ「AI判定失敗」になります）。

### Q. メールが送られてきません

いくつかの原因が考えられます。順番に確認してください。

1. キーワードに該当するメールが本当に届いているか、メールクライアントで確認してください
2. `config.json` の `keywords` の設定が正しいか確認してください
3. `config.json` の `smtp_host` / `smtp_port` / `notify_to` が正しいか確認してください
4. それでも届かない場合は、手順1を見直して手動実行を試してみてください

### Q. 実行すると英語のエラー（Traceback）が出て止まる

上記の想定内エラー以外の問題が起きています。エラーメッセージの内容をコピーして、次の作業依頼のときに一緒に伝えてください。

## 6. 覚えておいてほしいこと

- **メールの確認は自動です**: このツールが勝手に確認するため、あなた自身がメーラーで毎回確認する必要はありません
- **AI判定は参考値です**: Claude AIが判定した「緊急度」と「返信要否」は参考値です。最終的な判断はあなた自身で行ってください
- **実行するたびにAI利用料が少額発生します**: Claude APIの利用料が1回あたり数円程度かかります（年間で数百円程度の想定）
- **キーワードはいつでも追加・変更できます**: `config.json` の `keywords` を編集して保存するだけで、次回実行から新しいキーワード設定が反映されます
