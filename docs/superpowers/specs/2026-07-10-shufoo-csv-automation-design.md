# SHUFOO掲載用CSV・画像ZIP作成の自動化 設計書

作成日: 2026-07-10

## 背景・目的

SHUFOOへのチラシ掲載申請には、店舗ごとに1行の登録用CSV（店舗ID・掲載期間・チラシタイトル・
画像ファイル名などを含む固定フォーマット）と、参照している画像ファイルをまとめたZIPの
2点が必要。

現状は、前回分のCSVをコピーして日付・タイトル・画像ファイル名だけを手作業で書き換えている。
店舗数が多く（現状は「お買い得チラシ」パターン17件、「やすい！」パターン13件）、書き換え漏れ・
ミスのリスクがあるため、対話形式のスクリプトで生成できるようにする。

対応するパターンは現状の2つ（お買い得チラシ／やすい！）で固定でよく、将来的な増加は想定しない。

## 実例CSVフォーマットの確認結果

実際に提供された `tirasi.csv`（お買い得チラシ）・`tirasiyasui.csv`（やすい！）は
cp932（Shift-JIS系）エンコードで、以下25列の固定フォーマット。

| # | 列名 | 内容 |
|---|---|---|
| 1 | チラシ入稿ID | 店舗ごとに決まっている登録番号（例: `20250221`）。新規登録時に付与され、以後は不変 |
| 2 | Shufoo!店舗ID(必須) | 店舗ごとの固定ID |
| 3 | 先方店舗名 | 店舗名 |
| 4 | 掲載開始日時(必須) | 例: `2026/7/9 19:00` |
| 5 | 掲載終了日時(必須) | 例: `2026/7/11 15:00` |
| 6 | チラシタイトル(必須) | 例: `お買い得チラシ7月2週目` |
| 7〜22 | チラシ画像ファイル（1〜16ページ目） | 現状は1ページ目のみ使用、2ページ目以降は空欄 |
| 23 | ページスタイル | 実例では`1`固定 |
| 24 | ページの開き方 | 実例では空欄 |
| 25 | 掲載期間表示有無 | 実例では`N`固定 |

画像ZIP（`7.10-11.zip`, `yasui7.10-12.zip`）にはCSVの「チラシ画像ファイル」列で参照されている
画像ファイルのみが格納されている。CSVとZIPは別々にSHUFOOへアップロードする運用であり、
1つのZIPにCSVと画像を混在させることはしない。

## 全体構成

`日次業務\shufoo\` フォルダに以下を新規作成する。

| ファイル | 役割 |
|---|---|
| `config.json` | パターンごとの店舗マスター（店舗ID・店舗名・チラシ入稿ID）と出力ファイル名 |
| `make_shufoo_csv.py` | 対話形式でCSVと画像ZIPを生成するメインスクリプト |
| `test_make_shufoo_csv.py` | CSV組み立て・ZIP生成ロジックのユニットテスト |

他の自動化ツール（`メール処理/`など）と同じく、設定値はconfig.jsonに集約し、コメント・ログ
メッセージは日本語で書く。

## config.jsonの構造

```json
{
  "patterns": {
    "otoku": {
      "label": "お買い得チラシ",
      "csv_filename": "tirasi.csv",
      "zip_filename": "tirasi_images.zip",
      "stores": [
        {"entry_id": "20250221", "store_id": "881827", "store_name": "お菓子のデパートよしや　ユニバーサルシティ駅前店"}
      ]
    },
    "yasui": {
      "label": "やすい！",
      "csv_filename": "tirasiyasui.csv",
      "zip_filename": "tirasiyasui_images.zip",
      "stores": [
        {"entry_id": "20251009", "store_id": "881840", "store_name": "やすい！桃谷店"}
      ]
    }
  }
}
```

初期値として、今回提供された2つのCSVに含まれる店舗一覧（お買い得17件・やすい13件）を
そのまま登録する。店舗の追加・削除・パターン間の入れ替えは、config.jsonを直接編集する
運用とする（今回のスクリプトでは店舗マスターの編集機能は持たない）。

## 対話フロー（`make_shufoo_csv.py` 実行時）

1. パターンを選択する（`1: お買い得チラシ` / `2: やすい！`）
2. 掲載開始日・掲載終了日を日付のみで入力する（例: `7/10`, `2026/7/10`）
   - 年を省略した場合は現在の年を使う
   - 掲載開始日時 = 「開始日の前日 19:00」を自動計算（全パターン共通のルール）。
     `datetime.date - timedelta(days=1)`で計算するため、月またぎ・年またぎ
     （例: 開始日が1/1の場合は前年12/31 19:00になる）も自動的に処理される
   - 掲載終了日時 = 「終了日 15:00」を自動計算（全パターン共通のルール）
3. チラシタイトルを入力する
4. 既定の画像ファイル名を入力する（そのパターンの全店舗に適用するデフォルト値）
5. 「画像が異なる店舗はありますか？」と確認する
   - ある場合、店舗一覧を番号付きで表示して選択し、その店舗専用の画像ファイル名を入力する
   - 空Enterが入力されるまで繰り返し、複数の例外店舗を登録できる
6. 最終的に使用される画像ファイル名（既定＋例外）が `日次業務\shufoo\` フォルダに
   実在するか確認する。存在しないファイルがあれば警告を表示し、続行するか確認する
7. CSVを生成し、そのパターンの`csv_filename`（例: `tirasi.csv`）に上書き保存する
   - 文字コードはcp932
   - 各店舗1行、25列。列1〜3はconfig.jsonの店舗マスターから、列4〜6は入力値、
     列7（1ページ目画像）は既定値または例外値、列8〜22は空欄、
     列23=`1`・列24=空欄・列25=`N`で固定
8. 生成したCSVで実際に使用された画像ファイル名を重複除去して集め、そのパターンの
   `zip_filename`（例: `tirasi_images.zip`）に上書き保存する
9. 完了メッセージとして、生成件数・例外店舗数・CSV/ZIPの保存先パスを表示する

## エラー処理

- 画像ファイルが`日次業務\shufoo\`フォルダに存在しない場合: 警告を表示し、続行するか
  ユーザーに確認する（提出前に気づけるようにする目的で、処理自体は止めない）
- 日付の入力形式が読み取れない場合: エラーメッセージを表示し、再入力を促す
- 選択したパターンの店舗リストが空の場合: エラーメッセージを表示して処理を終了する

## テスト方針

対話部分（`input()`によるユーザー入力）とCSV/ZIP組み立てロジックを分離し、ロジック側を
ユニットテスト可能にする。

- CSV行の組み立て: 既定画像の適用、例外店舗の画像差し替え、日時の自動計算
  （開始日の前日19:00・終了日15:00への変換）を検証する
- ZIP生成: 複数店舗が同じ画像ファイルを参照している場合に重複除去されることを検証する
- 画像ファイルの実在チェック: 存在するファイル・存在しないファイルそれぞれのケースを検証する

## 移行時の注意

- 今回提供された実例CSVの一部行では列23〜25（ページスタイル・ページの開き方・
  掲載期間表示有無）が空欄になっているものがあったが、これは手入力のばらつきとみなし、
  本自動化では常に`1`・空欄・`N`で統一して出力する
- 実例の「やすい！」パターンには掲載開始時刻が20:00になっている行があったが、
  これも手入力のばらつきとみなし、本自動化では全パターン共通で前日19:00に統一する
