# SNS（TikTok）投稿分析・企画ツール 設計書

## 背景・目的

お菓子のデパートよしや 天理店（駄菓子メインのお菓子販売店）のSNS業務を始めるにあたり、まず第一段階として「TikTok上の駄菓子・お菓子関連の人気投稿を分析し、投稿内容のアドバイスを得る」ツールを構築する。

投稿の自動化・効率化は次の段階として別途検討する（本設計のスコープ外）。

## 要件

- 対象SNS: TikTok
- 分析対象の投稿収集方法: ユーザーがTikTokアプリ・サイト上で手動で見つけた動画のURLリストを`config.json`に指定し、そのURLの動画詳細ページから自動でデータ取得する（**2026-07-04変更**: 当初はハッシュタグ・キーワードの自動検索を予定していたが、実装・実機確認の結果TikTokの検索ページはヘッドレスブラウザをボットとして検知しCAPTCHAでブロックすることが判明したため、検索は手動・データ取得のみ自動の方式に変更した）
- アドバイスの形式: 傾向レポート（人気ハッシュタグ・投稿時間帯・反応の良い投稿の特徴）＋ Claude APIによる具体的な投稿企画案の両方
- 実行頻度: 手動実行（`python run.py` を必要なときに自分で実行）
- 出力形式: Excelファイル

## 構成

```
SNS/
├── config.json          # 動画URLリスト・Claude APIキーなどの設定
├── scraper.py           # 動画URLから動画詳細ページのメタデータ収集
├── analyzer.py          # 収集データの集計（人気ハッシュタグ・投稿時間帯・上位動画など）
├── planner.py           # 傾向データをClaude APIに渡し、投稿企画案を生成
├── report.py            # 傾向データ＋企画案をExcelに出力
├── run.py               # 上記を順番に実行するエントリーポイント（手動実行）
└── data/
    ├── raw_YYYYMMDD.csv      # 収集した生データ
    └── reports/YYYYMMDD.xlsx # 出力レポート
```

## データフロー

1. ユーザーがTikTokアプリ・サイト上で気になる駄菓子・お菓子関連の動画を自分で探し、そのURLを `config.json` の `video_urls` に追加する
2. `python run.py` を手動実行する
3. `scraper.py` がSelenium（pos_systemのNetdoa NXログインと同じ技術）で各動画のURLを開き、キャプション・いいね数・再生数・投稿日などを取得し `data/raw_日付.csv` に保存する
4. `analyzer.py` がCSVを集計し、「よく使われるハッシュタグ」「反応が良い投稿の傾向」「投稿時間帯」などをまとめる
5. `planner.py` が集計結果をClaude APIに渡し、具体的な投稿企画案を数個生成する
6. `report.py` が傾向データ＋企画案をExcelファイル1枚にまとめて出力する（`data/reports/YYYYMMDD.xlsx`）

## 既知の制約・リスク

- **TikTok利用規約リスク**: 動画詳細ページへの自動アクセスもTikTokの利用規約に抵触する可能性がある。アカウントBANやIPブロックのリスクを抑えるため、アクセス1回ごとに待機時間を入れる・短時間に大量アクセスしないなどの配慮をscraper.pyに組み込む
- **ボット検知リスク**: 検索ページの自動巡回はTikTokのボット検知（CAPTCHA）でブロックされることを実機確認済み。そのため検索自体は手動とし、個別の動画詳細ページのみ自動アクセスする方式にしているが、動画詳細ページも将来的にブロックされる可能性はゼロではない
- **サイト構造変更に弱い**: TikTok側の画面構成が変わるとscraper.pyの修正が必要になる（pos_systemのNetdoa NX連携と同様の性質）
- **Claude API利用料**: `planner.py`実行のたびに少額のAPI利用料が発生する

## エラーハンドリング

- スクレイピングで一部の動画情報が取得できない場合は、その動画をスキップしてログに記録し、取得できた分だけで分析を続行する
- Claude API呼び出しが失敗した場合は、企画案なしで傾向データのみのExcelレポートを出力する（分析自体は止めない）

## 依存パッケージ

`selenium`, `openpyxl`, `anthropic`（Claude API用）— 既存の`seisan.py`と技術構成を合わせる

## テスト方針

TikTokサイトに依存する`scraper.py`部分は自動テストが難しいため、少数のキーワードで手動実行して動作確認する。集計ロジック（`analyzer.py`）はサンプルデータを使った自動テストを用意し、ロジックの正しさを担保する。

## スコープ外（将来の別プロジェクト）

- 複数SNS（Instagram, X, LINEなど）への対応
